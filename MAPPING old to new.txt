# SlackSpotter - Name Mapping Reference

## CORRECTED FRAME STRUCTURE

### Tab 1 Layout (LEFT SIDE):
```
TOP-LEFT:    SS_Tab1_RaidBuffCheckPanel      (Checkboxes + 2 announce buttons)
MIDDLE-LEFT: SS_Tab1_ProtectionPotionPanel   (6 prot pot buttons + List Everyone checkbox)
BOTTOM-LEFT: SS_Tab1_StatsPanel              (Stats + 3 buttons horizontal: Rep+, Rep-, Reset)
RIGHT-SIDE:  SS_Tab1_RaidListPanel           (Raid member table)
```

### Tab 2 Layout:
```
CENTER:      SS_Tab2_ShoutoutsFrame          (Colored chat UI - formerly ColorSend)
```

## Tab 2: Shoutouts Module (ColorSend Integration)

```

ColorSendMsgBox                        → SS_Tab2_Shoutouts_MessageBox

```

### Function Renaming
```
CS:Send()                              → SS_Shoutouts_Send()
CS:CreateUI()                          → SS_Shoutouts_CreateUI()

trim()                                 → SS_Shoutouts_Trim()
parseTags()                            → SS_Shoutouts_ParseTags()
(new) SS_ShowTab2Content()             → Shows Tab 2 UI
(new) SS_HideTab2Content()             → Hides Tab 2 UI
(new) SS_Shoutouts_HandleCommand()     → Handles /ss colored messages
(new) SS_Shoutouts_Initialize()        → Initializes module on load
```

## Notes for Development

1. **All new global functions/variables MUST use SS_ prefix**
2. **Tab-specific functions use format: SS_Tab[N]_[Panel]_[Element]_OnClick**
3. **Panel naming reflects FUNCTION not just elements (RaidBuffCheckPanel = checks + buttons)**
4. **Never pollute global namespace without SS_ prefix**
5. **When porting old functions, rename AND add to this document**
6. **Each panel is self-contained - can be hidden/shown independently**
